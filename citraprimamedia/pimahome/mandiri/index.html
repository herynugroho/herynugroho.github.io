⚠️ Kondisi Saat Ini

Alur Operasional Existing (Sebelum Integrasi API Bank)

Proses penagihan dan rekonsiliasi keuangan saat ini sangat bergantung pada intervensi manusia (*human intervention*), berpotensi memicu perlambatan (bottleneck) hebat terutama pada awal bulan saat tagihan membeludak.

FASE 1

Penagihan Manual

Sistem PINISI men-generate invoice, lalu admin mengirimkan pesan tagihan ke pelanggan secara manual via broadcast WhatsApp.

Langkah Lambat
FASE 2

Titik Kritis Bayar

Pelanggan mentransfer dana, namun wajib mengirimkan bukti transfer ke WA CS. Status pembayaran rentan mengambang (*floating*).

Risiko Tinggi
FASE 3

Verifikasi Mutasi

Tim Finance login ke portal bank, mengecek mutasi secara berkala, dan mencocokkan nominal. Rawan kesalahan pencatatan internal.

Sangat Rawan Error
FASE 4

Aktivasi Manual

Setelah divalidasi, Admin mengubah status di sistem billing menjadi lunas, lalu membuka isolir manual di dashboard ACS/Mikrotik.

Delay Layanan

Kesimpulan Hambatan: Sinkronisasi data dari transaksi bank ke pemulihan internet pelanggan membutuhkan waktu berkisar antara 15 menit hingga beberapa jam, bergantung penuh pada ketersediaan admin jaringan di jam kerja.

⚡ Target Solusi Integrasi

Alur Operasional Rencana (Mandiri Host-to-Host)

Dengan implementasi arsitektur **Mandiri Bill Collection**, intervensi manual dipangkas habis. Core Engine PINISI dan sistem Bank Mandiri akan saling bertukar data secara otomatis dan instan.

1. INVOICING

Request Nomor VA

Sistem PINISI mengkalkulasi tagihan, mengirim request nomor VA secara H2H, dan mengunci nilai nominal transaksi melalui mekanis **Close Payment**.

2. TRANSACTION

Multi Channel

Pelanggan membayar tagihan lewat ATM, Livin' Mandiri, ekosistem bank lain, maupun e-wallet mitra dengan nomor VA yang didapat secara instan.

3. CALLBACK

Notifikasi Real-Time

Sesaat setelah dana masuk, gateway Bank Mandiri langsung melempar notifikasi callback menuju server PINISI. Invoice lunas otomatis dalam hitungan milidetik.

4. PROVISIONING

Auto-Aktivasi

Sistem penagihan memicu perintah terotomatisasi (*API Script trigger*) ke server ACS/jaringan utama untuk membuka isolir koneksi internet pelanggan.

5. SETTLEMENT

Rekonsiliasi Otomatis

Bank Mandiri merampungkan sinkronisasi data laporan rekening koran berformat **MT940** harian secara terjadwal langsung via File Transfer/Email.

Hasil Akhir: Seluruh rantai pemrosesan transaksi hingga pembukaan isolir internet selesai dalam waktu **kurang dari 10 detik, berjalan otomatis penuh 24/7**, tanpa membutuhkan intervensi admin finance sama sekali.
